Pharmacy Benefit Coding

The Pharmacy Coding Execution team (PCE) works on Commercial Line of Business and is responsible for intaking work from upstream implementation coordinators and translating the intent of our groups, into appropriate benefit configuration within our benefit systems. Additionally, they are typically the first point of contact for clarification questions surrounding benefits from our up streams.

Pharmacy Benefit Definition

Pharmacy Benefit: RX benefit, or plan is defined as a list of items, services or features included or excluded under the pharmacy portion of the health insurance, specifying all details and terms of the RX coverage. A group or client can create multiple different benefit designs, as well as multiple groups or clients can share the same benefit design. “Pharmacy benefit deductible is separate from medical in majority of cases”

Objective

The objective of the Benefit Coding is to ensure that all Rx benefits to be configured to plan as per client requirement. Coding process includes Spider, Benefit Central, Benefit Builder and Pharmacy Benefit Quality (Audit). All these coding process intent is to code or configured and validate the given Rx benefits to the member plan.

Benefit Repositories

Benefit System Purpose
Benefit Central (BC) Benefit Central is a web-based application that PBM uses to administer pharmacy benefits. Pharmacy benefits determine all the features on how drug claims adjudicate when a member goes to a pharmacy to fill a prescription.
Spider (Strategic Product Information Data Enterprise Repository) is the Product and Benefits system used to house benefit information for small group individual and ACA exchange business.
Benefit Builder Houses RX benefit information for Large Group and National Accounts

Benefit Coding Upstream Systems

  • Sales Team: Sales team will assign the projects through COM2 and Jira. Team reviews the member request and send those requirements in the form of paperwork (SOB/SBC/CEP/GSF/ASF)

Benefit Coding Process Cycle

Process Flow: COM2 Assignment→ Create JIRA→ Coding→ Update Intake/CFAN→ Benefit Tracker→ Table Update or CCS Link or Account Structure update→ Close JIRA and COM2

Applications Used in Benefit Coding Configuration Process

  • COM2: COM2 (Commercial2) is a system that sales team update the group information and benefits and used to get the inventory for the group size less than 250 and create pending correction for any clarifications in COM2.
  • JIRA: JIRA is a tool developed by Atlassian. It is a web-based application and used to get the inventory for the group size greater than 250+ and also to create the sub task for any clarifications in Jira.
  • Benefit Tracker: It is tool that is used for auditing the Proxies.
  • QA Log: It is a web-based application used to determine the plan completion status.
  • CCS Linking: CCS linking is done once the proxy moved to production.
  • eCEM: Enterprise Customer Exception Management - eCEM is a document which has Exception listed for that particular group/states.
